# 3.1. User Stories 
En esta sección se verán las historias de usuario que ayudarán en el desarrollo de la plataforma **DomotiCore**, el consumo energético que se ve mediante tecnología IoT, bajo la visión de **Veltrix**.

| Epic / Story ID | Título | Descripción | Criterios de Aceptación | Relacionado con (Epic ID) |
| :--- | :--- | :--- | :--- | :--- |
| EP01 / US01 | Vinculación de inicio | **Como** Usuario (Hogar/SME), **quiero** registrar mi inicio en la plataforma **para** establecer el puente de comunicación entre mis dispositivos físicos y la nube de DomotiCore. | **Scenario 1:** Registro exitoso del Gateway. **Dado que** el usuario tiene un inicio de Veltrix conectado a la red, **cuando** ingresa el código (MAC/ID) y presiona "Vincular", **entonces** el sistema valida la conexión y muestra "Online". <br><br> **Scenario 2:** Error de conexión. **Dado que** el Gateway no tiene internet, **cuando** intenta vincular, **entonces** el sistema muestra "Dispositivo no detectado". | EP01: Gateway Management |
| EP01 / US02 | Registro y sincronización de nodos | **Como** Usuario, **quiero** añadir nuevos enchufes o focos inteligentes **para** ampliar mi red de control. | **Scenario 1:** Sincronización. **Dado que** el usuario activa modo emparejamiento, **cuando** escanea dispositivos, **entonces** el sistema detecta y registra el nodo. | EP01: Gateway Management |
| EP02 / US03 | Control remoto de dispositivos | **Como** Usuario, **quiero** encender o apagar dispositivos desde el dashboard **para** gestionarlos remotamente. | **Scenario 1:** Encendido/apagado. **Dado que** el usuario ve el switch, **cuando** presiona toggle, **entonces** el sistema envía la señal y cambia el estado en tiempo real. | EP02: Device Control |
| EP03 / US04 | Programación de horarios | **Como** Usuario de Hogar, **quiero** programar apagados automáticos **para** reducir consumo. | **Scenario 1:** Configuración semanal. **Dado que** accede a Automations, **cuando** define horario, **entonces** el sistema guarda y ejecuta la regla. | EP03: Automation |
| EP03 / US05 | Creación de escenas grupales | **Como** Dueño de Negocio, **quiero** activar una escena de cierre **para** apagar equipos rápidamente. | **Scenario 1:** Escena multidevice. **Dado que** hay dispositivos agrupados, **cuando** presiona la escena, **entonces** se apagan todos simultáneamente. | EP03: Automation |
| EP04 / US06 | Monitoreo en tiempo real | **Como** Usuario, **quiero** ver el consumo en Watts **para** entender mi gasto energético. | **Scenario 1:** Visualización. **Dado que** el dispositivo está activo, **cuando** accede al detalle, **entonces** se muestra una gráfica en tiempo real. | EP04: Energy Monitoring |
| EP04 / US07 | Reporte de ahorro mensual proyectado | **Como** Administrador, **quiero** ver reportes comparativos **para** validar ahorro. | **Scenario 1:** Reporte. **Dado que** hay datos por 30 días, **cuando** genera reporte, **entonces** el sistema compara consumo y calcula ahorro. | EP04: Energy Monitoring |
| EP05 / US08 | Alerta de consumo inusual | **Como** Usuario, **quiero** recibir alertas por consumo alto **para** detectar fallas. | **Scenario 1:** Alerta. **Dado que** hay límite definido, **cuando** se supera, **entonces** se envía notificación push. | EP05: Notifications |
| EP05 / US09 | Notificación de olvido en horario crítico | **Como** Usuario, **quiero** ser notificado **para** ver si un equipo sigue encendido fuera de horario. | **Scenario 1:** Detección. **Dado que** hay consumo fuera de horario, **cuando** se detecta >0W, **entonces** envía alerta. | EP05: Notifications |
| EP06 / US10 | Logeo de eventos y auditoría | **Como** Administrador, **quiero** ver historial de acciones **para** un control de seguridad. | **Scenario 1:** Historial. **Dado que** hubo cambios, **cuando** accede a logs, **entonces** ve tabla con timestamp, usuario y acción. | EP06: Security & Logs |
| EP07 / US11 | Visualización general del hogar o negocio | **Como** Usuario, **quiero** ver un resumen general de mis dispositivos **para** conocer rápidamente el estado de mi hogar o negocio. | Scenario 1: Vista general. **Dado que** el usuario ingresa al dashboard principal, **cuando** carga la información, **entonces** el sistema muestra cantidad de dispositivos activos, apagados y con alertas. | EP07: Dashboard Overview |
| EP07 / US12 | Identificación de dispositivos por ambiente | **Como** Usuario, **quiero** asignar dispositivos a ambientes **para** organizarlos mejor. | Scenario 1: Asignación. **Dado que** el dispositivo está registrado, **cuando** el usuario selecciona un ambiente, **entonces** el sistema guarda la relación correctamente. | EP07: Dashboard Overview |
| EP08 / US13 | Gestión de usuarios secundarios | **Como** Administrador, **quiero** agregar usuarios con permisos limitados **para** compartir el control de dispositivos de forma segura. | Scenario 1: Creación de usuario. **Dado que** el administrador ingresa datos válidos, **cuando** guarda el nuevo usuario, **entonces** el sistema asigna permisos según el rol definido. | EP08: User Management |
| EP08 / US14 | Edición de permisos de acceso | **Como** Administrador, **quiero** modificar permisos de usuarios **para** mantener el control del sistema. | Scenario 1: Actualización. **Dado que** existe un usuario registrado, **cuando** se actualizan sus permisos, **entonces** el sistema aplica los cambios inmediatamente. | EP08: User Management |
| EP09 / US15 | Visualización del historial de consumo | **Como** Usuario, **quiero** revisar el consumo pasado de mis dispositivos **para** analizar mis hábitos energéticos. | Scenario 1: Historial. **Dado que** existen datos almacenados, **cuando** el usuario selecciona un rango de fechas, **entonces** el sistema muestra el consumo correspondiente. | EP09: Consumption History |
| EP09 / US16 | Descarga de reportes básicos | **Como** Usuario, **quiero** descargar reportes de consumo **para** revisarlos o compartirlos. | Scenario 1: Descarga. **Dado que** el reporte está generado, **cuando** el usuario solicita la descarga, **entonces** el sistema entrega un archivo con la información. | EP09: Consumption History |
| EP10 / US17 | Información del producto en la landing page | **Como** Visitante, **quiero** conocer qué es DomotiCore **para** entender su propuesta de valor. | Scenario 1: Visualización. **Dado que** el visitante ingresa a la landing page, **cuando** navega a la sección principal, **entonces** el sistema muestra información clara del producto. | EP10: Website |
| EP10 / US18 | Formulario de contacto | **Como** Visitante, **quiero** enviar una consulta **para** obtener más información del servicio. | Scenario 1: Envío de formulario. **Dado que** el visitante completa los campos requeridos, **cuando** envía el formulario, **entonces** el sistema registra la consulta correctamente. | EP10: Website |
| EP11 / US19 | Autenticación básica de API | **Como** Developer, **quiero** autenticar las solicitudes a la API **para** proteger el acceso a los datos. | Scenario 1: Token válido. **Dado que** el request incluye un token válido, **cuando** se procesa la solicitud, **entonces** la API responde con estado exitoso. | EP11: RESTful API |
| EP11 / US20 | Consulta de estado de dispositivos vía API | **Como** Developer, **quiero** obtener el estado de los dispositivos **para** integrarlos con otras aplicaciones. | Scenario 1: Consulta exitosa. **Dado que** el request es válido, **cuando** se consulta el endpoint, **entonces** la API devuelve el estado actualizado del dispositivo. | EP11: RESTful API |
